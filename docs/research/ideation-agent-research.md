## 3. 方案设计

### 3.1 功能边界
**包含**：
- 头脑风暴（多角度思考、创意扩展）
- 写作辅助（大纲生成、段落优化）
- 翻译/润色（多语言、风格调整）
- 快速问答（基于 Memo 上下文的知识激发）

**不包含**（未来迭代）：
- 长文生成（超出 Agent 能力边界）
- 代码编程（Geek Agent 负责）

### 3.2 技术方案
**后端**：
- 新增 `config/parrots/ideation.yaml` 配置
- 复用 `report_inability` 工具
- 可选：新增 `memo_write` 工具（将创意写入笔记）

**前端**：
- 无需新页面，Chat 界面天然支持
- Prompt hints 引导用户使用

**AI 代理**：
- 使用 `direct` 策略（快准稳，不需要 ReAct 推理）
- 系统 prompt 定义创意生成 Protocol
- 可注入 Memo 上下文作为素材

### 3.3 用户体验
**主要流程**：
1. 用户输入创意相关请求
2. Orchestrator 路由到 Ideation Agent
3. Agent 生成创意内容
4. 可选：写入 Memo 保存

**边缘情况**：
- 请求超出能力 → 使用 report_inability

### 3.4 风险与缓解
| 风险 | 影响 | 缓解措施 |
| :--- | :--- | :------- |
| LLM 输出质量不稳定 | 中 | 明确 Output Format 约束 |
| 与 Memo 边界模糊 | 低 | 定义「生成」vs「搜索」的差异化 |
