# Orchestrator 通用子代理调研报告

> 日期: 2026-02-17 | 版本: v1.0 | 状态: ✅ 已创建 Issue #252

## 需求概述

**用户 Idea**：Orchestrator 是否应该具备"通用智能"，在不需要专家 Agent 的领域，直接处理简单任务。

### 需求拆解

| 需求 | 描述 |
|------|------|
| 通用任务直接响应 | 总结、翻译、简单问答等场景无需调用专家 |
| 场景判断 | "明显能判断"的场景走通用路径 |

### 用户场景

- 输入1: "最近记了什么笔记" → 调用 Memo 搜索
- 输入2: "总结这些笔记" → 应直接使用上一轮结果，LLM 摘要

## 调研结论

### 技术方案

采用 **方案 C（直接返回）** 作为 MVP：

1. **TaskPlan 扩展**：新增 `DirectResponse` 字段
2. **Decomposer Prompt 扩展**：新增判断规则和 few-shot
3. **Executor 分支处理**：支持直接响应路径

### 触发场景

| 场景 | 示例 |
|------|------|
| 总结归纳 | "总结这些笔记"、"概括一下" |
| 翻译 | "翻译成英文" |
| 简单问答 | "今天星期几" |
| 格式转换 | "把列表改成表格" |

### 工作量评估

- **工作量**：0.5 人天
- **风险**：中
