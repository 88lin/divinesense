# 国际化规范 (i18n)

## 核心规则

- **所有 UI 文本必须使用 `t("key")`** —— 禁止硬编码字符串
- 翻译 key 必须同时存在于 `en.json` 和 `zh-Hans.json`
- 验证命令：`make check-i18n`

## 翻译文件位置

- 英文：`web/src/locales/en.json`
- 简体中文：`web/src/locales/zh-Hans.json`

## Key 命名规范

使用点分隔的层级命名：
- `pages.home.title` — 页面标题
- `components.button.submit` — 组件文本
- `errors.network.timeout` — 错误消息

## 添加新 Key 步骤

1. 在组件中使用 `t("your.new.key")`
2. 同时添加到 `en.json` 和 `zh-Hans.json`
3. 运行 `make check-i18n` 验证
